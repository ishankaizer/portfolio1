# Documentation

Long-term source of truth for this project. Start with
[`../CLAUDE.md`](../CLAUDE.md) for the short version and the hard rules.

| Document | What it covers |
|---|---|
| [architecture.md](./architecture.md) | Stack, rendering model, routing, data layer, build pipeline, deploy |
| [design-language.md](./design-language.md) | "The Working Drawing", the philosophy-over-motif rule, tokens, typography, layout |
| [animation.md](./animation.md) | Motion principles, the reveal system, the hero intro, and the robustness rules |
| [conventions.md](./conventions.md) | TypeScript, component, styling, accessibility, and asset conventions |
| [decisions.md](./decisions.md) | Enduring decisions with reasoning, plus paths tried and rejected |
| [content.md](./content.md) | Editing copy, adding a project, case study shape, drop-in assets |
| [known-gaps.md](./known-gaps.md) | Verified dead code, stale references, and inconsistencies |

## Reading order

- **New to the repo:** `../CLAUDE.md`, then `architecture.md`, then `design-language.md`.
- **Changing how something looks:** `design-language.md`, then `conventions.md`.
- **Touching motion:** `animation.md`, and read the robustness rules before writing code.
- **Wondering why something is the way it is:** `decisions.md`.
- **Only updating content:** `content.md` is all you need.
